# 🧾 Freelancer Invoice Manager

A private, self-hosted PHP invoice manager for freelancers. Login-protected, PDF-generating, currency-switching, and built for one user to deploy on shared hosting in minutes.

**Built with:** PHP 8.4 · MySQL · DomPDF · Vanilla JS · Lora + DM Mono

---

## Features

- 🔐 Secure single-user login (Argon2id hashing, CSRF protection, rate limiting)
- 📋 Create, edit, delete invoices with dynamic line items (qty × rate = live total)
- 💱 Per-invoice currency switching between € EUR and $ USD — updates everywhere instantly
- 📄 Downloadable branded PDF invoices via DomPDF
- 📊 Dashboard with status tracking (Draft → Sent → Paid → Overdue)
- 🏦 Bank / IBAN / BIC payment details on every PDF
- 🇩🇪 Built-in Kleinunternehmerregelung VAT notice (§ 19 UStG) — easily removed or edited
- 🎨 Branded design system (yellow / teal / red palette, Lora + DM Mono typography)

## Deployment (Apache / Shared Hosting)

### 1. Upload files
Upload all contents to your subdomain or domain document root.

### 2. Create the database
In phpMyAdmin or MySQL CLI, create a database and run `sql/schema.sql`.

### 3. Configure environment
```bash
cp .env.example .env
```
Edit `.env` with your DB credentials, site URL, and freelancer details.

### 4. Generate your password hash
```bash
php setup/generate-password.php
```
Paste the output hash into `.env` as `APP_PASSWORD_HASH`.
**Delete `setup/generate-password.php` after use.**

### 5. Install dependencies
```bash
composer install --no-dev --optimize-autoloader
```

### 6. About fonts
Lora is the default font used and the following fonts can be found in `assets/fonts/`:
```
Lora-Regular.ttf
Lora-Italic.ttf
Lora-SemiBold.ttf
Lora-Bold.ttf
Lora-BoldItalic.ttf
```
> DomPDF cannot load fonts from Google's CDN — local TTF files are required for PDF rendering hence they are included with this project.

### 7. Set folder permissions
```bash
chmod 755 storage/font-cache/
chmod 755 storage/pdf-cache/
```

## Environment Variables

See `.env.example` for the full list. Key entries:

| Key | Description |
|---|---|
| `SITE_URL` | Full URL, no trailing slash: `https://invoices.your-website.com` |
| `APP_USERNAME` | Your login username |
| `APP_PASSWORD_HASH` | Argon2id hash generated by `setup/generate-password.php` |
| `DB_*` | MySQL connection credentials |
| `FREELANCER_*` | Your name, address, email, bank details — appear on all PDFs |

## Customisation

### German VAT notice
The § 19 UStG notice appears on every invoice as this is a requirement for German invoicing for freelancers making under a certian amount per year. To remove or change it, edit:
- `includes/invoice-form.php` — the web form
- `templates/invoice-pdf.php` — the PDF output

### Brand colours
All colours are CSS custom properties at the top of `assets/css/style.css` and you can edit instances of the below across project files to customize colors (there is probably an easier way to do this):
```css
:root {
    --yellow: #FAC946;
    --teal:   #007998;
    --red:    #F90002;
    --dark:   #333333;
    --body:   #404040;
}
```

## Security

- Passwords hashed with `PASSWORD_ARGON2ID`
- Login rate-limited to 5 attempts / 15 minutes
- All DB queries use PDO prepared statements
- All output escaped with `htmlspecialchars()`
- CSRF tokens on every POST form
- Session cookies: HttpOnly, Secure, SameSite=Strict
- Sensitive directories blocked via `.htaccess`
- `.env` blocked from web access via `.htaccess`

## Known Limitations

- Single-user only — no registration or multi-user support
- No email sending — download PDF and send manually
- No client portal — invoices are internal only
- No recurring invoices — create manually each period
- Colors are hard coded in PHP files so may require editing of the instances in those files to customize them.  Sorry!
